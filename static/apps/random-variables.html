Random Variable Visualizer

pre-alpha

An interactive take on the idea that a random variable is a quantity that fluctuates as its underlying randomness ω is resampled. Write a short, Excel-like program — X = Unif({1..6}), Y = Exp(3), Z = X + Y, condition on … — and watch the values, running statistics, and animated Monte-Carlo plots update live. There is one shared ω per tick, so every variable is drawn together and coupling and independence come out exact and automatic.

Work in progress. This is a pre-alpha: the app and its little language are still evolving, and behaviour may change or break between versions. Feedback is welcome — and the language reference at the foot of the page lists what works today.

A companion to Terence Tao's 2016 blog post “Visualising random variables”, whose animated-GIF illustrations this app makes interactive; several features were suggested by that post's commenters. Runs entirely in your browser. Coded with the assistance of Claude Code.

Language reference

  
The app runs a small, Excel-like expression language. Each line is one statement, and
    # begins a comment. Everything listed here is supported today; this reference will grow
    as the language does.


  
The model — one shared ω

    
A random variable is stored as an expression over independent primitive
      draws, not as a fixed number. Every tick a single shared sample point ω is drawn
      and all primitives are resampled together, so:

    

      
Z = X + Ystays coupled to X and Y (same ω)

      
W = X - Xis exactly 0, always

      
two separate drawsare independent automatically

    

    
Reusing a name refers to the same variable; writing a fresh
      Distribution(...) makes a new, independent one.

  


  
Assignments, constants & comments

    

      
Name = exprdefine a random variable

      
let name = exprdefine a constant — a fixed number, used for parameters and vector sizes

      
a == bequality test (gives 1 or 0); a single = is assignment

      
:=accepted as an alias for =

      
# …comment to the end of the line

    

    
Conventions (recommended, not enforced): Capitalised names for
      random variables, lower-case for constants, UPPERCASE for functions.

  


  
Distributions

    

      
Unif(a, b)continuous uniform on the interval [a, b]

      
Unif({…})discrete uniform on a set, e.g. Unif({1..6}) (a die)

      
Exp(rate)exponential with rate λ (mean 1/λ)

      
Normal(μ, σ)Gaussian, mean μ and standard deviation σ (also Gaussian(μ, σ))

      
Bernoulli(p)1 with probability p, otherwise 0

      
Binomial(n, p)number of successes in n independent p-trials

      
Poisson(λ)Poisson counts with mean λ

      
Geometric(p)trials up to and including the first success; support {1, 2, 3, …}, mean 1/p

    

  


  
Sets (for discrete uniforms)

    

      
{1..6}the integer range 1, 2, …, 6

      
{1, 2, 5, 10}an explicit set of values

    

  


  
Operators & comparisons

    

      
+   −   *   /   ^arithmetic (^ is power); unary minus too

      
<   <=   >   >=   ==   !=comparisons — each gives a 0/1 indicator random variable

    

    
Example: I = (X > 3) is 1 when X exceeds 3, else 0; its mean estimates P(X > 3).

  


  
Functions

    
Reductions (vector → scalar)

    
SUM, PROD, AVERAGE (= MEAN), COUNT,
      STDEV, VAR, MAX, MIN.

    
Elementwise maths

    
ABS, SQRT, EXP, LOG (= LN),
      SIN, COS, TAN, FLOOR, CEIL,
      ROUND, SIGN.

    
Logic

    
IF(condition, a, b) — a when the condition holds, else b.

    
MAX and MIN also accept several scalar arguments, e.g. MAX(X, Y, 0).

  


  
Vectors & the central limit theorem

    

      
Sample(dist, n)a length-n vector of independent draws from dist (n is a constant)

    

    
Combine with a reduction to watch concentration in action:

    

      
let n = 100

      
S = (SUM(Sample(Unif(0,1), n)) − n/2) / SQRT(n/12)

      
plot Sa standardised sum → approaches a standard normal as n grows

    

  


  
Commands: plot & condition

    

      
plot X1-D view (fading dots + histogram)

      
plot X, Y2-D scatter

      
several plot linesopen several plot windows at once (stacked)

      
condition on Erejection sampling: keep only the ticks where the 0/1 condition E holds. It conditions the shared ω for every variable, so dependence can appear (e.g. an independent square collapses into a disk).

    

    
If a condition is very unlikely, the app caps the number of tries per tick; the
      acceptance rate is shown under the Values panel.

  


  
The plot panel & theory overlay

    
Controls: style (dots / histogram / both), fuzz (spreads
      coincident points so density shows), colours (highlight the current sample, or a
      continuous gradient trail), 1:1 aspect lock for 2-D, and theory.
      When theory is on and the plotted variable is a single primitive distribution, its exact
      pdf/pmf is drawn over the histogram; for derived variables it reads “unavailable”
      rather than guessing.
